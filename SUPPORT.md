# Getting Support

Need help with Azimuth? Here's where to look, in order.

## Documentation first

- **[README](README.md)** — install, shortcuts, and command behavior.
- **[User manual](https://ai-screams.github.io/Azimuth/manual.html)** — the full keyboard
  reference and how each command works.
- **[FAQ](https://ai-screams.github.io/Azimuth/#faq)** — common questions (permissions,
  multi-monitor, size-increment apps like Terminal).

Most questions — "why is there a gap after maximizing?", "how do I remap a shortcut?", "how do I
grant Accessibility?" — are answered there.

## Still stuck?

- **Bug or unexpected behavior:** open an
  [issue](https://github.com/ai-screams/Azimuth/issues). Please include your Azimuth version
  (**Azimuth → About**), macOS version, and clear reproduction steps.
- **Feature idea:** open an issue describing the problem you want solved (Azimuth favors
  predictable, composable commands — see the README design principle).

## Common first checks

- **Shortcuts do nothing?** Make sure Accessibility is enabled in
  **System Settings → Privacy & Security → Accessibility**, and check the Settings window — a
  combo shown as *"In use by system"* is claimed elsewhere and can be remapped.
- **Running an old version?** Azimuth updates in place via Sparkle — use **Check for Updates…**
  from the menu bar or Settings.

## Please don't use issues for

- **Security vulnerabilities** — follow [`SECURITY.md`](SECURITY.md) to report privately.

## Supporting the project

Azimuth is free and open source, and always will be. If you'd like to fund its development, see
the **Support** section of the [README](README.md#support) — entirely optional, always appreciated.
